How this works: On startup, the Rust backend calls [[NSUserDefaults standardUserDefaults] setBool:NO forKey:@"..."] for every key variation before the WKWebView is created. This is a 100% public, App Store-safe API. The FPS counter above uses requestAnimationFrame to measure the actual frame rate delivered by the webview. If any key successfully disables the 60fps cap, you should see FPS values above 60 on a ProMotion display. Toggle individual keys to isolate which one (if any) has an effect. Note: toggling keys at runtime may not take effect until the webview is recreated (restart the app after changing).
